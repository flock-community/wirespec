# Architecture Menu — Move `intro-ir.md` Out of `intro/`

**Date:** 2026-05-08
**Status:** Approved

## Problem

`src/site/docs/docs/intro/intro-ir.md` is a deep dive into the Wirespec compiler's IR pipeline (Convert → Transform → Generate). It sits next to `intro-contract.md`, `intro-validate.md`, and `intro-generate.md`, which are user-facing introductions explaining what Wirespec is and how to use it. The IR page is compiler-internals content and does not fit that audience. A reader following the intro menu top-to-bottom is dropped into language-neutral IR trees and transformer DSLs without any onboarding bridge.

## Goal

Create a new top-level docs menu named **Architecture** for technical/internals content, and move `intro-ir.md` into it. Leave the rest of the intro menu focused on user onboarding.

Scope is intentionally minimal: move one file, add a stub index page. No content rewrite, no doc split, no other technical pages migrated.

## Non-Goals

- Rewriting the IR doc or splitting it into multiple pages.
- Writing a substantive Architecture overview index. The index is a stub for now and grows later.
- Migrating other technical content (e.g., from `CLAUDE.md`) into the new menu.
- Restructuring any other docs folder.

## Detailed Changes

### 1. New file: `src/site/docs/docs/architecture/architecture.md`

Stub index — short, no padding. Frontmatter sets the menu's slot in the sidebar.

```markdown
---
title: Architecture
sidebar_position: 10
---

# Architecture

Deep dives into how Wirespec works internally — the compiler pipeline, the
intermediate representation, and how target-language code is generated.
```

`sidebar_position: 10` places the menu at the end of the sidebar, after all user-facing menus (intro, getting-started, language, emitters, plugins, integration, converters, ide). Architecture is "go deep" content; bottom-of-sidebar matches that intent.

### 2. Move + reframe: `intro/intro-ir.md` → `architecture/architecture-ir.md`

The file move is a `git mv` so history is preserved. Content unchanged.

Frontmatter changes:

| Field             | Before                | After                       |
|-------------------|-----------------------|-----------------------------|
| `title`           | `IR Model`            | `IR Model` (unchanged)      |
| `sidebar_position`| `4`                   | `2`                         |
| `slug`            | (absent)              | `/architecture/ir`          |

`sidebar_position: 2` puts the IR page directly under the new index (position 1). The added `slug` matches the convention used by `plugins/` and `emitters/` files (e.g., `slug: /plugins/cli`, `slug: /emitters/java`), giving a clean URL `/architecture/ir`.

### 3. Nothing else changes

- `sidebars.ts` is unchanged — it autogenerates from the folder structure.
- No `_category_.json` is added; the index page's frontmatter handles ordering and the title.
- The `intro/` menu's remaining pages keep their existing positions.

## Verification

After applying the changes, in `src/site/docs/`:

1. `npm run build` (or `yarn build`) — Docusaurus build must succeed with no broken-link warnings.
2. Manually inspect the dev sidebar (`npm run start`): "Architecture" appears at the bottom, expands to "IR Model".
3. Open `/architecture/ir` — page renders with the full IR Model content, no 404.
4. Confirm `/intro/intro-ir` no longer exists (or redirects naturally via Docusaurus' missing-page behavior — no redirect rule is required since no inbound links exist).

## Risks / Notes

- **Inbound links:** Verified none in repo. External links (blog posts, social) to `/intro/intro-ir` would break. Acceptable: the page is recent and a 404 is preferable to a stale path; redirects can be added later if traffic data justifies it.
- **Folder ordering:** Multiple existing folders share `sidebar_position: 1` on their index pages, so Docusaurus' alphabetical fallback decides their order. Setting `architecture` to `10` is unambiguous and won't shuffle other folders.
- **Future growth:** If the menu grows to 3+ pages, revisit whether the stub index needs to become a real overview. Out of scope for this change.
